Machine Failure Detection using a Wide & Deep ANN Model

TensorFlow Keras SciKeras Scikit-Learn TensorBoard Feature Engineering Matplotlib/Seaborn Pandas NumPy

Predictive maintenance on industrial sensor data (Kaggle): end-to-end pipeline with hash-based splits, a Keras Wide+Deep classifier, SciKeras + RandomizedSearchCV tuning, and TensorBoard monitoring—delivering ~98.7–99.1% test accuracy.

▼

I built a predictive-maintenance model on the Machine Predictive Maintenance Classification dataset (10,000 rows) using a Wide & Deep neural network in Keras. The pipeline includes hash-stable train/val/test splits (70/15/15), structured preprocessing (median imputation, scaling, one-hot encoding), and hyperparameter tuning with RandomizedSearchCV (SciKeras). With BatchNorm, L1/L2 regularization, EarlyStopping, ReduceLROnPlateau, and TensorBoard tracking, the model achieved ~98.7% overall test accuracy (best epoch eval ~99.1%, test loss ~0.044–0.056).

- Data scale: 10,000 rows, ~9 model inputs (6 numeric features + 1 categorical one-hot to 3 columns).
- Split strategy: Hash-based partition on unique ID for reproducible 70/15/15 train/val/test.
- Preprocessing: Median imputation, standardization for numeric; OneHotEncoder for categorical; label handling via pipelines.
- Model: Keras Wide & Deep (dense tower + direct wide inputs) with BatchNormalization and L1/L2 regularization.
- Training controls: EarlyStopping (restore best), ReduceLROnPlateau, TensorBoard logs for loss/accuracy curves.
- Tuning: SciKeras + RandomizedSearchCV over depth/width and regularization/learning-rate.
- Results: Validation accuracy peaked ≈ 98.5%; test accuracy ≈ 98.7% overall with a best epoch evaluation ≈ 99.1% (test loss ≈ 0.044–0.056).
- Outline: Seaborn EDA (distributions, box/violin plots, pair plots, correlations) → cleaning & unit conversions → feature prep → baseline → tuned model.

Used Car Price Prediction using Scikit-Learn and Streamlit

Random Forest Decision Tree Linear Regression TensorFlow Keras SciKeras Scikit-Learn TensorBoard Joblib Streamlit

Developed an end-to-end regression pipeline to predict U.S. used car prices using 200 K+ real listings. The project emphasizes model interpretability and user interaction, featuring a tuned Random Forest model served via a Streamlit web app.


[ VISIT MY STREAMLIT ]

▼

This project predicts used car prices from over 200,000 real U.S. vehicle listings using Scikit-Learn pipelines and a tuned RandomForestRegressor. After extensive EDA, data cleaning, and feature engineering, the final model achieved an R² of ~0.95 on validation data. Key features such as year, mileage, horsepower, and make were visualized with feature importance charts to highlight interpretability. The entire pipeline was deployed through Streamlit, allowing users to input car details and obtain instant price predictions.

- Dataset: 200 K rows × 20+ features (mileage, horsepower, year, make, engine type, etc.) from Kaggle’s US Used Cars Dataset (Ananay Mital).
- Preprocessing: Handled missing data, converted date/time features, standardized numerics, and applied OneHotEncoder for categoricals via ColumnTransformer.
- EDA: Distribution plots, outlier removal (price/mileage/day-on-market), and correlation checks using Matplotlib and Seaborn.
- Model: RandomForestRegressor optimized with RandomizedSearchCV and cross-validation for best generalization.
- Performance: Achieved R² ≈ 0.95 and low MAE, demonstrating strong predictive power and low error spread.
- Explainability: Visualized feature importances—showing top drivers like year, mileage, and horsepower—to communicate insights clearly.
- Deployment: Packaged final pipeline into an interactive Streamlit app where users can explore predictions and model logic in real time.

Traffic Sign Recognition via Deep CNN (TensorFlow + OpenCV)

OpenCV CNN TensorFlow Keras Scikit-Learn SciKeras TensorBoard Seaborn

Built a custom CNN in TensorFlow to classify 43 German traffic sign categories from the GTSRB dataset. Applied OpenCV preprocessing (resize, normalize, one-hot encode) and trained using Conv-BN-LeakyReLU-Dropout blocks, achieving ~98–99% accuracy with TensorBoard and SciKeras tuning for performance tracking and optimization.

▼

This deep learning project classifies 43 categories of German traffic signs using a custom Convolutional Neural Network (CNN) built from scratch in TensorFlow/Keras. Over 50,000+ images were preprocessed with OpenCV for resizing, normalization, and one-hot label encoding. The model architecture combines Conv-BN-LeakyReLU blocks, Dropout regularization, and dense output layers, achieving ~98–99% accuracy on held-out data. Training was monitored with TensorBoard and improved through EarlyStopping, ReduceLROnPlateau, and RandomizedSearchCV (SciKeras) hyperparameter tuning, highlighting both robust performance and engineering maturity.

- Dataset: ~50,000 labeled images (43 classes) from the GTSRB – German Traffic Sign Benchmark.
- Preprocessing: Loaded with OpenCV, resized to 64×64×3, normalized (0–1), and one-hot encoded labels.
- Architecture:
  - Conv2D → BatchNorm → LeakyReLU → MaxPooling → Dropout (×3 blocks) → Flatten → Dense(128–256) → Dropout → Softmax(43).
- Regularization: BatchNormalization, Dropout (0.3–0.5), and data-driven early stopping.
- Training: Adam optimizer; EarlyStopping, ReduceLROnPlateau, and TensorBoard monitoring of loss and accuracy curves.
- Tuning: SciKeras + RandomizedSearchCV explored hyperparameters (dropout rate, activation, learning rate, BN toggles).
- Results: ~99% training accuracy, ~98% validation accuracy, cross-entropy loss ≈ 0.04–0.05.
- Visualization: TensorBoard dashboards showing convergence and performance per epoch; Seaborn and Matplotlib used for class distribution and sample visualization.

End-to-End Grocery Reorder Classifier (SQL + SciKeras Pipeline)

SQL (sqlite3) TensorFlow Keras Scikit-Learn SciKeras Pandas NumPy Matplotlib Seaborn

Developed an end-to-end ML pipeline to predict product reorders using the Instacart dataset (~3.4M orders). Combined SQL feature engineering and a Keras neural network (via SciKeras) tuned with RandomizedSearchCV, achieving strong precision–recall results across millions of user–product interactions.

▼

This project predicts whether a user will reorder a product using the Instacart Online Grocery dataset of over 3.4 million transactions. I built a complete data-to-model pipeline—merging six relational tables in SQLite, engineering user–product features such as recency, frequency, and reorder rate, and preprocessing them with Scikit-Learn pipelines. The model, a Keras neural network wrapped in SciKeras, was optimized through RandomizedSearchCV and trained with BatchNorm, Dropout, and early-stopping callbacks, delivering strong generalization and reproducible results.

- Dataset scale: ~3.4M orders, 49K products, 200K users from Kaggle’s Instacart Online Grocery Dataset.
- Data engineering: Built a consolidated SQL feature table (orders × products × users × departments × aisles).
- Feature engineering: Derived recency, frequency, days_since_prior_order, cart_position, and reorder ratio per user–product pair.
- Preprocessing: Encoded categorical variables, scaled numerics, and implemented reproducible train/val/test splits using hash-based partitioning.
- Modeling: Constructed a Keras neural network integrated into a Scikit-Learn pipeline via SciKeras, optimized using RandomizedSearchCV.
- Training: Regularized with BatchNorm, Dropout, EarlyStopping, ReduceLROnPlateau; monitored via TensorBoard.
- Evaluation: Measured Precision@K, Recall@K, and AUC to assess reorder prediction quality.
- Outcome: Delivered an efficient, reproducible pipeline demonstrating expertise in data wrangling, feature design, and applied ML engineering.
